# 毕业设计整改任务清单

## 必须立即修改

1. 问题：云端后端当前访问超时，前端只能打开静态页面，无法保证上传识别流程。  
   修改目标：答辩前必须能稳定完成“上传图片 -> 后端处理 -> 返回预览和代码”。  
   建议做法：检查 Render 启动命令、服务入口、环境变量和依赖；至少保证 `/`、`/docs` 可访问；准备本地启动备用方案。  
   影响位置：部署 / 代码 / PPT。

2. 问题：README 后端启动命令错误，写的是 `uvicorn main:app`，但 FastAPI `app` 在 `backend/main_api.py`。  
   修改目标：保证指导老师或答辩检查人员能按说明跑起来。  
   建议做法：改成实际可用命令，例如在项目根目录运行 `uvicorn backend.main_api:app --reload`，并说明前端如何配置后端地址。  
   影响位置：说明文档 / 部署。

3. 问题：论文和 PPT 写了 OCR、文本识别、PaddleOCR 或 OCR 准确率，但本地代码没有 OCR 实现。  
   修改目标：论文、PPT、代码一致。  
   建议做法：若来不及实现 OCR，就把 OCR 改成“后续扩展”或“未实现的预留方向”；同时删除 OCR 准确率。若保留，则补依赖、代码、接口返回和测试证据。  
   影响位置：论文 / PPT / 代码 / 查非项 / AI味。

4. 问题：PPT 第 12 页写 YOLO 检测准确率 >85%、OCR >90%、单页生成时间 <15 秒、支持并发请求处理，但当前没有证据。  
   修改目标：答辩展示只保留可证明的数据。  
   建议做法：删除无证据指标，或补真实测试表：测试图片、模型、耗时、返回结果、指标计算方法、截图。  
   影响位置：PPT / 论文 / 查非项。

5. 问题：第 6 章测试章节偏形式化，只有“测试通过”，缺少真实输入输出和日志。  
   修改目标：把测试章改成能支撑毕业设计可信度的证据章节。  
   建议做法：至少补 5 个测试样例，写明图片文件名、尺寸、选择模型、返回字段、生成耗时、SSIM/MSE/PSNR、失败或回退情况。  
   影响位置：论文 / AI味 / 查非项。

6. 问题：PPT 技术栈与代码不一致，写了 Element Plus、Axios、PaddleOCR、DeepSeek-VL。  
   修改目标：PPT 中的技术栈必须和 `package.json`、`requirements.txt`、后端代码一致。  
   建议做法：改为 Vue3、Vite、fetch、FastAPI、YOLO、OpenAI-compatible API、Qwen/DeepSeek/豆包；没有实现的不要写。  
   影响位置：PPT / 论文一致性。

7. 问题：后端返回检测图 URL 写死 `http://127.0.0.1:8000/data/...`，云端前端会无法访问。  
   修改目标：云端检测图能正常展示。  
   建议做法：返回相对路径 `/data/xxx_detect.jpg`，或根据请求 Host 生成完整线上 URL。  
   影响位置：代码 / 部署。

8. 问题：论文抽取文本中存在“重庆邮电大学硕士学位论文第1章引言论文结构”等模板残留。  
   修改目标：消除模板污染和不符合本科毕业设计身份的内容。  
   建议做法：用 Word/WPS 全文搜索“硕士”“论文结构”“PAGE \\* MERGEFORMAT”“SHAPE \\* MERGEFORMAT”，检查页眉页脚、隐藏域和末尾空白内容。  
   影响位置：论文 / 查非项。

## 建议尽快修改

1. 问题：结构还原度模块在论文中写得较复杂，但代码实际主要返回整体 SSIM/MSE/PSNR。  
   修改目标：论文描述与代码真实能力一致。  
   建议做法：要么补灰度、边缘、结构 SSIM、像素 SSIM 分离逻辑；要么把论文改为“基于截图相似度的辅助评价”。  
   影响位置：代码 / 论文 / PPT / AI味。

2. 问题：布局树论文称“最近父节点”，代码实际可能把子节点挂到更外层父框。  
   修改目标：避免实现描述和代码逻辑不一致。  
   建议做法：修改论文表述为“基于包含关系建立层级”，或改代码选择面积最小的包含父框。  
   影响位置：代码 / 论文。

3. 问题：`.env.example` 使用 `DOUBAO_API_KEY`，代码读取 `ARK_API_KEY`。  
   修改目标：豆包模型配置能按说明工作。  
   建议做法：统一变量名，并在 README 写清楚 `ARK_BASE_URL`、`DOUBAO_MODEL` 等配置。  
   影响位置：代码 / README / 部署。

4. 问题：前端上传地址写死为 Render 地址，本地调试不方便。  
   修改目标：支持本地和线上环境切换。  
   建议做法：使用 `VITE_API_BASE_URL`，本地 `.env.development` 指向 `http://127.0.0.1:8000`，线上指向 Render。  
   影响位置：代码 / 部署 / README。

5. 问题：参考文献 [3]、[28] 未在正文引用中检出，且引用顺序不严格。  
   修改目标：满足参考文献和引用规范。  
   建议做法：删除未引用文献或在正文补充合理引用；按学校要求核对是否按首次出现顺序排列。  
   影响位置：论文 / 查非项。

6. 问题：图编号格式混乱，如 `图 4-2`、`图 4.2`、`图3.7` 混用。  
   修改目标：图表编号和题注格式统一。  
   建议做法：统一为学校模板要求的格式，逐图检查正文是否先引用再放图。  
   影响位置：论文 / 查非项。

7. 问题：论文 AI味较强，尤其是摘要、绪论、需求分析、测试、总结。  
   修改目标：让论文更像本人真实系统开发记录。  
   建议做法：减少“能够、基本、一定程度、可以看出、测试通过”，增加具体代码路径、接口字段、测试截图、日志和失败分析。  
   影响位置：论文 / AI味。

8. 问题：PPT 第 10 页运行效果展示不够实，只列“原始截图、预览、代码、布局树”。  
   修改目标：答辩时一眼能看到系统成果。  
   建议做法：放真实系统截图，包含上传图、生成预览、代码片段、布局树 JSON、检测图。  
   影响位置：PPT。

9. 问题：`frontend/README.md` 仍是 Vue/Vite 模板说明。  
   修改目标：归档材料专业统一。  
   建议做法：替换为前端运行说明、环境变量、构建命令、目录说明和常见问题。  
   影响位置：说明文档。

10. 问题：项目缺少独立用户手册和部署说明。  
    修改目标：答辩前检查和归档更完整。  
    建议做法：新增简短文档，写清系统使用流程、模型配置、云端部署、常见失败原因和备用演示方案。  
    影响位置：说明文档 / 部署。

## 可选优化

1. 问题：规则生成兜底页面较占位化，叶子节点基本都是按钮。  
   修改目标：兜底结果更像 UI 原型。  
   建议做法：根据 bbox、role、宽高比例区分按钮、输入框、卡片、图片占位、标题。  
   影响位置：代码 / 论文。

2. 问题：无自动化测试文件。  
   修改目标：提高工程可信度。  
   建议做法：补 pytest 测试布局树、HTML 清洗、模型返回解析；前端至少补构建检查或关键函数说明。  
   影响位置：代码 / 论文测试。

3. 问题：演示依赖外部模型接口，答辩现场不稳定。  
   修改目标：降低现场网络和 API 风险。  
   建议做法：准备 2-3 个已生成结果的离线样例，提供本地 JSON/HTML 回放模式。  
   影响位置：代码 / 部署 / PPT。

4. 问题：测试图片文件扩展名与真实格式部分不一致，如 `.jpg` 文件实际为 PNG。  
   修改目标：归档材料更规范。  
   建议做法：统一文件扩展名，按 `test_login.png`、`test_dashboard.png` 这类语义命名。  
   影响位置：代码材料 / 论文测试。

5. 问题：PPT 页码显示“1/14”但实际共 15 页。  
   修改目标：答辩材料细节更专业。  
   建议做法：统一页码，删除重复页或改为 15 页编号。  
   影响位置：PPT。

6. 问题：前端仍保留未使用的 Vite 模板组件和样式。  
   修改目标：减少项目模板痕迹。  
   建议做法：删除未使用的 `HelloWorld.vue` 或说明其未使用；清理默认 Vite 文案样式。  
   影响位置：代码 / AI味。

7. 问题：README 目录结构列出实际不存在或被忽略的文件，如 `GeneratedLayout.vue`、`preview.html`。  
   修改目标：说明文档与仓库一致。  
   建议做法：标注这些是运行时生成文件，不是仓库固定文件。  
   影响位置：README。

8. 问题：论文摘要和结论中“具有一定应用价值”“提升效率”等结论偏泛。  
   修改目标：降低 AI味，提高可信度。  
   建议做法：改为“在测试样例中可完成原型级页面生成；复杂 UI 还原仍需人工调整”。  
   影响位置：论文 / AI味。

9. 问题：PPT 创新点“提高生成质量”证据不足。  
   修改目标：避免答辩被追问无法证明。  
   建议做法：改为“增强生成过程可解释性，便于调试和展示”。  
   影响位置：PPT。

10. 问题：论文部分 2026 年参考文献存在时效核验风险。  
    修改目标：避免引用未正式发表或信息不准的文献。  
    建议做法：逐条联网核验题名、作者、期卷页、出版状态，必要时替换为已正式发表文献。  
    影响位置：论文 / 查非项。
